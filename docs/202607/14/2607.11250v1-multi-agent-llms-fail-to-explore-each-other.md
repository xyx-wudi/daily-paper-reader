## 摘要
探索对于多智能体系统的可靠自主性至关重要，但尚不清楚大语言模型（LLM）智能体在相互交互时能否有效探索。我们表明，现代LLM智能体无法做到这一点，它们常常表现出短视和两极分化的交互模式，导致次优协调和遗憾增加。我们将这一挑战正式定义为多智能体探索问题，将其建模为部分可观察随机博弈（POSG）问题，其中智能体必须探测同伴以推断其能力并识别有效的交互策略。为了解决这个问题，我们引入了多智能体上下文探索（MACE），这是一个轻量级框架，通过结构化同伴选择明确促进探索。在上下文多样性和参数多样性设置中，MACE显著改善了探索行为和下游任务性能。我们还从理论上证明，探索的价值随着智能体多样性的增加而增加。总体而言，我们的结果揭示了当前LLM智能体的一个基本局限性，并强调了明确引导探索对于可靠多智能体自主性的重要性。代码将在https://github.com/deeplearning-wisc/mace发布。

## Abstract
Exploration is essential for reliable autonomy in multi-agent systems, yet it remains unclear whether large language model (LLM) agents can explore effectively when interacting with one another. We show that modern LLM agents fail to do so, often exhibiting myopic and polarized interaction patterns that lead to suboptimal coordination and increased regret. We formalize this challenge as the Multi-Agent Exploration problem, modeling it as a partially observable stochastic game (POSG) problem in which agents must probe peers to infer their capabilities and identify effective interaction strategies. To address this, we introduce Multi- Agent Contextual Exploration (MACE), a lightweight framework that explicitly promotes exploration through structured peer selection. Across both contextual and parametric diversity settings, MACE substantially improves exploration behavior and downstream task performance. We further show theoretically that the value of exploration increases with agent diversity. Overall, our results highlight a fundamental limitation of current LLM agents and underscore the importance of explicitly guided exploration for reliable multi-agent autonomy. Code will be released in https://github.com/deeplearning-wisc/mace

# 论文总结：Multi-Agent LLMs Fail to Explore Each Other

## 1. 核心问题与整体含义（研究动机与背景）

当前大语言模型（LLM）越来越多地被部署为多智能体系统中的自主代理，它们需要相互通信、委托任务并做出决策。然而，论文发现现代LLM代理在与其他代理交互时**无法有效探索**——它们表现出短视和两极分化的交互模式（过早锁定某个同伴），导致次优协调和累积遗憾增加。这一问题在能力或上下文存在异质性的多代理环境中尤为严重。论文将此挑战形式化为**多智能体探索问题**，将其建模为部分可观测随机博弈（POSG），并证明缺乏显式探索机制是当前多代理LLM系统可靠自主性的根本障碍。

## 2. 方法论：Multi-Agent Contextual Exploration (MACE)

- **核心思想**：将每个代理的同伴选择问题分解为独立的**上下文bandit**问题。通过关系特征表征交互的非平稳性和结构，并利用LinUCB算法中的乐观原则显式驱动探索。
- **关键技术细节**：
  - **特征设计**：对每个候选同伴构建9维关系特征向量，包括：响应多样性（1-3 gram Jaccard多样性）、同伴独特性（网络层面响应差异度）、历史平均奖励、归一化交互轮次、偏置项。
  - **奖励函数**：混合奖励 = 0.5 × (改进分数 + 最终绝对分数)，其中改进分数 = 当前轮次F1/准确率减去上一轮。
  - **选择规则**：\( a_{i,t} = \arg\max_a \left[ \hat{\theta}_{i,a}^\top \mathbf{x}_{i,a,t} + \alpha \sqrt{\mathbf{x}_{i,a,t}^\top \mathbf{A}_{i,a}^{-1} \mathbf{x}_{i,a,t}} \right] \)，其中\(\mathbf{A}_{i,a}\)是累积设计矩阵，\(\hat{\theta}_{i,a}\)为岭回归估计。
  - **更新**：每轮选择同伴后更新对应\(\mathbf{A}, \mathbf{b}\)矩阵，再重新计算\(\hat{\theta}\)。
- **理论保证**：MACE达到\(O(\sqrt{T \log T})\)次线性遗憾，而贪婪策略（无探索）可能产生\(\Omega(\delta T)\)线性遗憾（\(\delta\)为能力多样性）。探索优势随代理多样性增长。

## 3. 实验设计

- **异质性设置**：
  - **上下文多样性**：HotpotQA（distractor模式），10个Qwen2.5-7B-Instruct代理，每个看到不同的上下文片段（2个相关+8个干扰，随机分配）。交互轮次R=5。
  - **参数多样性**：Math500（级别4-5）和GPQA（Diamond split），4个代理：GPT-5, Qwen2.5-7B-Instruct, Llama3.1-8B-Instruct, Mistral-7B-v0.3。交互轮次R=3。
- **基线方法**：
  1. In-Context Exploration：代理仅通过提示获取相同信息（历史、特征），自主选择。
  2. Random：均匀随机选择同伴。
  3. Pre-defined：固定选择右邻代理（类似多数多代理辩论结构）。
- **评估指标**：Exact Match（EM）和F1（HotpotQA），准确率（Math500, GPQA）；累积遗憾（基于最优同伴的期望奖励差）。
- **两个阶段**：前一半样本用于**试错阶段**（参数学习），后一半用于**利用阶段**（MACE参数冻结，其他基线保持原有策略）。另外进行到未见过的任务2WikiMultihopQA的**迁移实验**。
- **额外实验**：探索系数\(\alpha\)敏感性分析；更强同质模型组（GPT-5, GPT-4, GPT-5.4-mini, GPT-5.4-nano）；完全同质代理（5个Qwen2.5-7B）；MACE的时序差分扩展MACE-TD。

## 6. 主要结论与发现

1. **LLM代理天生缺乏探索能力**：在简单两臂赌博机实验中，GPT-4、GPT-5、Qwen2.5-7B均表现出过早极化选择，而非平滑积累证据。In-Context Exploration甚至不如随机选择同伴。
2. **MACE显著改善探索与性能**：在所有设置中，MACE的累积遗憾低于所有基线，且在利用阶段（参数冻结）和迁移任务中持续领先。明确的结构化探索优于纯提示驱动。
3. **探索价值随代理多样性增加**：理论（Corollary 1）和实验（同质设置下收益较小）共同证实：当代理能力/信息高度异质时，探索收益更大；当代理几乎同质时，探索的额外价值有限。
4. **学习策略可迁移**：在HotpotQA训练的参数可直接用于2WikiMultihopQA，且仍优于从头探索的基线，表明MACE捕获了通用交互结构。

## 7. 优点

- **问题形式化**：首次将多代理LLM探索问题严格建模为POSG，并定义了能力多样性与后悔下界。

- **实验设计严谨**：区分试错/利用两阶段，验证策略的可迁移性；消融和敏感性分析全面；使用多个前沿模型确保结论的一般性。

- **规模有限**：实验中代理数量最多10个（上下文多样性）或4个（参数多样性），未评估数百/数千代理的大规模场景。在这种规模下通信瓶颈、动态反馈等问题可能更突出。

- **未讨论通信成本与鲁棒性**：实验假设代理间通信无成本且安全，实际部署中可能有延迟、噪声、恶意攻击等挑战。
