## Abstract 

Multi-agent planning becomes substantially harder when agents must improve specialized decision-making skills while keeping their inter-nal implementations private. This regime arises when agents are developed independently, expose different interfaces and capabilities, and must nev-ertheless coordinate without sharing executable policies. Prior research has largely assumed cen-tralized optimization, shared policy access, or common skill representations, making it poorly suited to privacy-constrained cooperation. We introduce RELIC, a framework for learning inter-pretable and composable skills via revealed prin-ciples. Each agent refines its own programmatic skill through private LLM-guided search, while a trusted orchestrator evaluates proposed updates solely through team-level performance. Success-ful behaviors are not broadcast as code; instead, they are abstracted into portable principles that other agents can instantiate within their own in-terfaces and recombine with local strategies. This separates coordination from implementation shar-ing, enabling cross-agent transfer under heteroge-neous skill signatures. RELIC thus introduces a new paradigm for privacy-preserving skill learn-ing and coordination in multi-agent planning.

In parallel, a growing line of work studies skill discovery and skill libraries, where LLMs or offline data are used to acquire reusable behaviors that can later be retrieved, adapted, or composed (Zhang et al., 2023; Zhu et al., 2026; Wang et al., 2026; Alzubi et al., 2026; Li et al., 2026). These directions have established that LLMs are effective at producing structured planning artifacts and reusable skills. However, most existing approaches assume that planning knowledge is improved in a centralized form: a shared plan-ner, a common symbolic model, or a globally accessible skill repository. This assumption is limiting for multi-agent planning settings in which different agents expose different decision interfaces, specialize in different roles, and must still improve as a team. In such settings, the key challenge is not merely to optimize each skill independently, but to discover transferable coordination structure : what aspects of a successful local policy can be abstracted, communi-cated, and re-instantiated by another agent with a different interface and role. We study multi-agent planning problems in which each agent is represented by its own programmatic decision skill, implemented as an interpretable function with an agent-specific signature. The agents must improve jointly as a team, yet candidate updates are evaluated only through over-all team performance. This creates a coupled search prob-lem: a local change should be retained only if it improves coordination at the team level, while any useful behavioral structure should remain transferable across agents with dif-ferent interfaces. We introduce RELIC ( RE vealed Principles for Learning 

Interpretable Composable Skills), a framework for learning in this setting. Each agent performs LLM-guided search over revisions of its own skill program, while a trusted orchestrator evaluates candidate updates under the cur-rent team and maintains the best joint solution. Rather than transferring successful skills directly as implementa-tions, RELIC distills them into revealed principles : com-pact descriptions of strategies or coordination patterns that can be re-instantiated under another agent’s local signa-ture. Con-cretely, RELIC combines private per-agent program search, a shared public tree of revealed principles, and operators for principle instantiation and reflection. This perspective differs from prior multi-agent LLM sys-tems that focus on communication protocols, centralized planning, or retrieval from pre-existing skill libraries (Li et al., 2025; Yang et al., 2025; Li et al., 2026). Our goal is instead to learn transferable coordination principles jointly with the underlying skills, under heterogeneous agent in-terfaces and team-level supervision. This makes RELIC well suited to teams of independently specialized agents that must develop interoperable behavior. This work makes three contributions. First, we formulate a new setting for multi-agent planning: learning interoperable programmatic skills under heterogeneous agent interfaces and team-level evaluation. Second, we propose RELIC, a framework that combines private skill search with principle-mediated transfer to learn interpretable and composable be-haviors. Third, across diverse multi-agent planning domains, we show that RELIC improves both team performance and cross-agent skill reuse over competitive baselines.

2.2. LLM-Guided Program Tree Search 

RELIC builds on LLM-guided program search methods such as MCTS-AHD (Zheng et al., 2025) and MOTIF (Kiet et al., 2026), which represent search directly in code space. The root node contains a seed skill, and each child node is obtained by applying an LLM-guided editing operator to its parent program. Each node stores a candidate skill π, together with standard tree-search statistics such as visit count N (π) and value estimate Q(π). Baselines. We compare RELIC against six baselines. GS

HAOS (Pei et al., 2024) uses a learned gate to switch between offline and online hyper-heuristic selec-tors. By contrast, RELIC refines skills in a decentralized manner at the agent level while handling streaming batches under centralized orchestration. ## A. Related Works 

We briefly review the most relevant research threads surrounding our setting, with particular emphasis on LLM-based heuristic construction, reusable skill learning, and decentralized cooperation. This discussion clarifies the gap addressed by RELIC: interoperable skill discovery across independently trained agents under limited information sharing. 

A.1. LLMs for Automated Heuristic Design 

Recent work has shown that large language models (LLMs) can serve as powerful search operators for automated heuristic design (AHD), where the goal is to discover high-quality algorithmic components with minimal human intervention. A representative line of work is EoH (Liu et al., 2024), which combines evolutionary search with LLM-based generation of both high-level heuristic ideas and executable code, demonstrating that natural-language “thoughts” can be treated as evolving search objects alongside programs. ReEvo (Ye et al., 2024) further develops this direction by framing LLMs as reflective hyper-heuristics, using iterative self-reflection and evolutionary improvement to generate stronger heuristics and meta-heuristics across optimization tasks. Moving beyond population-based evolution, MCTS-AHD (Zheng et al., 2025) replaces greedy population updates with Monte Carlo Tree Search, preserving a richer search history and enabling more comprehensive exploration of the heuristic space. More recent work expands the design space beyond a single heuristic. MOTIF (Kiet et al., 2026) formulates automated solver design as a multi-strategy optimization problem, where multiple interdependent components are improved through a turn-based interactive framework between LLM agents. EoH-S (Liu et al., 2026) shifts the objective from discovering one universal heuristic to constructing a complementary heuristic set , aiming for stronger robustness across heterogeneous instances and distributions. HiFo-Prompt (ChentongChen et al., 2026) introduces hindsight and foresight mechanisms to guide LLM-based AHD, explicitly controlling the exploration–exploitation trade-off while accumulating reusable design principles over time. Overall, these methods primarily focus on centralized heuristic or component search. In contrast, RELIC studies cooperative planning under decomposition, where multiple agents improve private decision functions separately while exchanging only abstract principles through a shared public structure. 

A.2. LLMs for Planning Optimization 

A related line of work uses LLMs for automated planning and optimization in domains such as routing, itinerary generation, urban operations, and robot task planning. One common paradigm is to use LLMs as an interface between natural-language requests and downstream optimization modules. For example, TTG (Ju et al., 2024) translates user requests into symbolic constraints and then invokes a mixed-integer optimization solver to produce guaranteed travel plans. ItiNera (Tang et al., 2024) similarly combines LLM-based request understanding with explicit spatial optimization for open-domain urban itinerary planning. At a broader systems level, UrbanLLM (Jiang et al., 2024) studies LLMs as decision-making agents for urban activity planning and management, covering planning problems with realistic operational structure and constraints. These works show that LLMs are useful not only for language understanding, but also for generating structured optimization inputs and coordinating solver-based planning pipelines. Another closely related direction focuses on robotic planning , where LLMs are combined with symbolic actions, affordances, or motion feasibility constraints. SayCan (ichter et al., 2023) grounds language-model action selection using robot affordances, enabling long-horizon execution of high-level tasks on real robots. Inner Monologue (Huang et al., 2022) extends this idea by incorporating environment feedback and re-planning during execution. ProgPrompt (Singh et al., 2023) represents robot plans as executable program-like prompts, improving situated task planning under explicit action and object vocabularies. More recent systems push further toward hierarchical and embodied planning: DELTA (Liu et al., 2025) decomposes long-horizon robot tasks before invoking automated planners, while LLM-GROP (Zhang et al., 2025) integrates visual grounding with task-and-motion planning for mobile manipulation. Compared with these approaches, RELIC is not aimed at directly producing a single plan for one instance. Instead, it learns and improves reusable planning decision functions for cooperative multi-agent optimization, with private local search and restricted information sharing through distilled public principles. 8RELIC: Revealed Principles for Learning Interpretable Composable Skills in Multi-Agent Planning 

## B. Problems & Algorithms 

This appendix presents the six benchmark optimization problems considered in this work. They are organized into two families. The first family, multi-agent path planning , concerns teams of mobile agents that must coordinate their movements on a metric graph under limited travel budgets. These models are directly relevant to applications such as persistent monitoring, environmental reconnaissance, search-and-rescue, and field servicing. The second family, multi-agent scheduling , concerns decentralized dispatching over multiple resources or processing stages, where each agent controls a local decision rule while performance is evaluated through a global system-level objective. Such settings arise naturally in manufacturing, service operations, warehouse control, and heterogeneous human–robot teamwork. Throughout this appendix, the set of agents is denoted by 

K = {A, B, C }.

Each agent controls one local decision function. The resulting problems are cooperative: actions are selected locally, but the objective is defined at the team level.

## C. Experimental Settings 

Evaluation protocol. We evaluate all methods on the six benchmark problems described in the previous section. Each problem uses the same three-agent setup, with one local decision function per agent. For every problem, training consists of a stream of five batches with 10 instances per batch. The training size is smaller than the test size: the three path-planning problems are trained on instances with N = 40 nodes, the job-dispatch and skilled-task problems are trained on instances with N = 30 , and the flow-shop problem is trained on instances with N = 25 . Evaluation is conducted on two larger held-out test sets with N ∈ { 50 , 80 }, each containing 20 instances. RELIC processes the training stream sequentially in batch order. In contrast, all non-LLM baselines are trained in a centralized regime. GS, HHS, and CCEA operate on the aggregated offline training set; HAOS and RL-IS retain the batch decomposition but still train offline over multiple passes through the full stream rather than adapting in a single pass. For the heuristic-based baselines, the candidate primitive set is the same handcrafted role-specific pool used throughout the repository. For MOTIF, we match the problem interfaces, seed initialization, and search budget to those of RELIC to ensure a fair comparison. 

RELIC configuration. Unless otherwise stated, all RELIC experiments use the default configuration in the released implementation. We use gpt-4o-mini as the underlying LLM backbone with sampling temperature 1.0. At each training batch, every agent performs 10 private search iterations, yielding 5 × 3 × 10 = 150 private program-expansion steps per run. The private search tree uses maximum depth 8, replay size 2, and equal exploration constants for node, operator, and public-principle selection. 

Agent roles and function signatures. Across all benchmarks, the three agents follow the same high-level role decomposi-tion. Agent A is a myopic local decision maker: it observes only its own state and the currently available options. Agent B

is teammate-aware: it receives limited runtime information about other agents, such as current positions, busy times, or local states. Agent C is global-planning oriented: it receives a denser system-level representation, such as full remaining-value vectors, machine-load vectors, or pipeline state. This design creates heterogeneous skill signatures even within the same cooperative task. RELIC must therefore learn interoperable behavior across agents whose executable functions expose different arguments and information.
