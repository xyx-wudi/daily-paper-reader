## 摘要
视觉推理需要整合分布在不同区域、属性和关系中的证据，这使得单链推理容易过早产生感知承诺和幻觉。我们提出了Visual Para-Thinker++，一个单策略多智能体框架，其中共享的多模态大模型策略被实例化为角色条件化的主智能体、工作者智能体和总结智能体。主智能体通过固定的分配模式分解任务；工作者智能体在上下文隔离下并行推理；总结智能体协调完整的工作者推理轨迹，而不是对最终标签进行多数投票。共享策略通过多智能体能力注入和角色解耦的多智能体优化进行训练，为相应的令牌片段分配角色特定的奖励和优势，以减少协作角色间的梯度冲突。一个原生推理引擎通过共享视觉前缀和键值缓存重用实现了高效的多智能体推理。在V*、CountBench、RefCOCO系列和HallusionBench上，Visual Para-Thinker++一致优于单轨迹和推理时并行基线，在对幻觉敏感的视觉推理任务上尤其有显著提升。

## Abstract
Visual reasoning requires integrating evidence distributed across regions, attributes, and relations, making single-chain reasoning prone to early perceptual commitment and hallucination. We propose Visual Para-Thinker++, a single-policy multi-agent framework in which one shared MLLM policy is instantiated as role-conditioned Main, Worker, and Summary Agents. The Main Agent decomposes the task with fixed allocation patterns; Worker Agents reason in parallel under context isolation; and the Summary Agent reconciles full Worker reasoning traces rather than majority-voting on final labels. The shared policy is trained by Multi-Agent Capability Injection and Role-Decoupled Multi-Agent Optimization, which assign role-specific rewards and advantages to corresponding token segments to reduce gradient conflict among collaborative roles. A native inference engine enables efficient multi-agent rollout through shared visual prefix and KV cache reuse. Across V*, CountBench, the RefCOCO family, and HallusionBench, Visual Para-Thinker++ consistently outperforms single-trajectory and inference-time parallel baselines, with especially strong gains on hallucination-sensitive visual reasoning.

# Visual Para-Thinker++：用于视觉推理的单策略多智能体框架 — 详细总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

视觉推理需要整合分布在不同区域、属性和关系中的证据。传统的单链推理（single-chain reasoning）容易在早期产生“感知承诺”（perceptual commitment）和幻觉：一旦解释被选择，延长链条只会加深而非多样化该解释，早期感知错误往往被合理化而非纠正。为此，作者提出一种新的扩展维度——通过多角色协作而非延长单链来提升视觉推理能力。核心动机是：**不同推理角色（任务分解、局部证据收集、全局计数验证、中间发现协调）可以自然地分配给不同智能体**，从而弥补单一轨迹的不足。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想：单策略多智能体（Single-Policy Multi-Agent）

- 使用**一个共享的多模态大模型策略** πθ，通过角色令牌（role tokens）和可见上下文将其条件化为三个角色智能体：
  - **主智能体**（Main Agent）：根据图像和问题，通过两种固定任务分配模式（块基分配或扫描顺序分配）将任务分解为若干子问题，分发给工作者智能体。
  - **工作者智能体**（Worker Agents）：在上下文隔离下并行执行各自的子任务（每个工作者只能看到主智能体的输出和自己的推理，不可见其他工作者）。
  - **总结智能体**（Summary Agent）：读取所有工作者智能体的完整推理轨迹，进行**痕迹级证据协调**（trace-level evidence reconciliation），而非仅对最终标签进行多数投票，从而生成最终答案。

### 2.2 关键技术细节

- **多智能体能力注入（Multi-Agent Capability Injection, MAC）**：第一阶段通过角色感知的有监督微调（SFT）训练共享策略，公式为：
  ```
  L_MAC = -∑ log πθ(主片段) - ∑∑ log πθ(工作者i片段) - ∑ log πθ(总结片段)
  ```
  使用上下文隔离掩码（context-isolation mask）确保工作者独立，总结智能体可访问所有工作者痕迹。

- **角色解耦的多智能体优化（Role-Decoupled Multi-Agent Optimization）**：第二阶段使用组相对强化学习（基于DAPO），引入角色特定的奖励和优势：
  - **工作者奖励** R_worker：通过跨工作者多数投票启发式评估局部证据准确性。
  - **结果奖励** R_out：根据总结智能体的最终答案是否与真实标签一致。
  - **角色解耦优势**：对结果奖励和每个工作者奖励分别进行组归一化，得到A_out和A_worker。然后将优势按令牌组合：主和总结令牌用A_out，工作者i令牌用A_out + λ A_worker(i)。这避免了不同角色间的梯度冲突。

- **两种任务分配模式**：
  - **块基分配**：图像分为K个不相交区域（默认四象限），适合接地、细粒度感知和属性验证。
  - **扫描顺序分配**：每个工作者保留全局视野但按不同扫描顺序遍历图像（左→右、右→左、上→下、下→上），适合计数和全局验证。

- **原生多智能体推理引擎**：基于vLLM，通过视觉前缀共享和KV缓存写时复制（Copy-on-Write）实现高效推理。只需一次预填充图像-问题-主片段，分离K个工作序列并行解码，再合并KV页面供总结智能体使用。

### 2.3 算法流程（文字描述）

1. 输入图像和问题，主智能体根据任务类型（默认块基，若为计数/全局验证则切换为扫描顺序）生成任务分配模式m。
2. K个工作智能体在上下文隔离下并行推理，各自输出推理轨迹s_i。
3. 总结智能体读取所有s_i以及m，进行痕迹级协调，生成最终答案u。
4. 训练时：第一阶段MAC（SFT）学习角色特定能力；第二阶段RDMAO（DAPO）用角色解耦的优势（A_out和A_worker）更新共享策略。

## 8. 不足与局限

- **主干规模限制**：主要实验基于3B骨干，7B仅作参考，未在更大模型（如30B+）上验证。扩展行为未刻画。
- **基准覆盖有限**：未涵盖文档理解、图表推理、视频决策等更广泛的多模态场景。
- **主智能体规划简单**：仅使用两种固定分配模式（块基和扫描顺序），非自适应开放规划器。何时切换模式、改变工作者数量等未模型化。
- **工作者奖励启发式**：使用跨工作者多数投票作为工作者奖励，当大多数工作者共享同一幻觉时可能失效。
- **总结奖励仅基于最终答案**：未显式监督总结智能体的协调过程忠实性，未来需更强的过程监督。
- **效率仍有成本**：尽管优化，四工作者配置仍比单模型增加约2倍延迟（312s vs 266s），且无法避免推理总令牌的增加。
- **缺乏总结智能体独立消融**：未将训练好的总结智能体（痕迹级协调）与非学习性的多数投票进行对比，未能分离“痕迹阅读”与“证据协调”的贡献。
- **额外标注依赖**：第一阶段的SFT数据由更强教师合成，可能引入教师风格和偏见，且数据过滤规则可能过于保守。
- **伦理风险**：更自信的多智能体系统可能产生更有说服力的错误输出，不能替代人工审查。
