## 摘要
强化学习（RL）在序贯决策中取得了优异表现，但由于奖励稀疏、状态-动作空间庞大以及难以学习协调策略，将其扩展到复杂多智能体环境仍具挑战。我们提出一种分层架构，其中预训练的大语言模型（LLM）作为集中式策略控制器，为智能体团队选择专门的RL技能策略，而RL策略则负责反应式底层执行。我们在竞争性2v2 King of the Hill环境中，将此混合系统与行为树（BT）和“扁平”RL（无技能分解的端到端训练）基线进行了评估。LLM+RL系统取得了与手工BT统计等价的性能（胜率46.4% vs 51.5%，p=0.103），而两者均显著优于无技能分解训练的扁平RL。一项用户研究（n=15）显示，60%的参与者认为LLM+RL智能体最接近人类行为（p=0.027），并指出其行为适应性和战术多样性。这些结果表明，预训练LLM的推理能力可有效协调预训练RL技能，实现具有竞争力的多智能体协调和更佳的可信度感知，而无需手动规则工程。

## Abstract
Reinforcement learning (RL) has achieved strong performance in sequential decision-making, yet scaling to complex multi-agent environments remains challenging due to sparse rewards, large state-action spaces, and the difficulty of learning coordinated strategies. We propose a hierarchical architecture where a pretrained large language model (LLM) acts as a centralized strategic controller that selects among specialized RL skill policies for a team of agents, while RL policies handle reactive low-level execution. We evaluate this hybrid system in a competitive 2v2 King of the Hill environment against behavior tree (BT) and \emph{``Flat''} RL (end-to-end training without skill decomposition) baselines. The LLM+RL system achieves task performance statistically equivalent to hand-crafted BT (46.4\% vs 51.5\% win rate, $p=0.103$) while both significantly outperform Flat RL trained without skill decomposition. A user study ($n=15$) reveals that 60\% of participants perceive LLM+RL agents as the most human-like ($p=0.027$), citing behavioral adaptability and tactical variability. These results demonstrate that pretrained LLM reasoning can effectively orchestrate pretrained RL skills, achieving competitive multi-agent coordination and superior perceived believability without manual rule engineering.

### 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想**：采用**双层层次化控制架构**，将“战略规划”与“战术执行”解耦。
    *   **高层（战略层）**：一个预训练的LLM（Gemma 3 27B）作为集中式元控制器。
    *   **底层（执行层）**：一组预训练的RL技能策略，每个策略专注于单一的战术目标（如导航、战斗、固守、撤退）。
*   **关键技术细节**：
    1.  **架构流程**：
        *   在决策时间步（约0.5秒/次），LLM接收全局游戏状态（转为自然语言描述），为每个智能体从技能库中选择一个技能。
        *   每个智能体执行被分配的RL技能，该技能以高频（12.5Hz）将局部观测映射为低层动作（移动、旋转、跳跃、射击）。
    2.  **状态与观测的不对称性**：
        *   **LLM输入（全局状态）**：自然语言描述，包含联合团队状态（双方位置、血量、到目标的距离等）、游戏时间、进度等。状态变量被离散化为类别标签（如血量：高/中/低），以便进行语义推理。
        *   **RL策略输入（局部观测）**：20维的归一化数值向量，包含智能体自身相关的距离、角度、射线探测等信息。

            *   `Combat`（战斗）：攻击可见敌人。
            *   `Secure`（固守）：在目标区域内防守。
            *   `Retreat`（撤退）：在低血量时寻找血包。
        *   每种技能策略都使用PPO算法，在针对其战术目标设计的简化场景中进行独立训练，使用不同的奖励函数（如Navigate奖励接近目标，Combat奖励造成伤害）。
    4.  **LLM提示工程**：
        *   包含静态系统提示（游戏规则、技能使用指南、团队协作原则）和动态用户提示（当前的实时游戏状态）。LLM输出格式被严格限制为两个数字（如“1，2”），代表为两个智能体选择的技能ID。

### 3. 实验设计：场景、基准、对比方法

*   **场景**：一个定制的 **2v2 King of the Hill（占山为王）** 竞技环境。两支队伍争夺一个随机出现的“目标区域”，累计占据8秒则获胜。
*   **基准（Baselines）**：
    1.  **Behavior Tree (BT)**：一个手工设计的、基于优先级的规则系统，作为性能上限的参考。
    2.  **Flat RL**：一个端到端训练的单一PPO策略，不进行任何技能分解，作为无层次化的RL基线。
*   **对比方法**：**LLM + RL**（本文提出的方法）。
*   **评估方式**：
    1.  **性能评估**：三种智能体类型（LLM+RL, BT, Flat RL）之间两两对决，每种对阵进行**1000**场episode，共**3000**场。衡量指标包括胜率、击杀/死亡比（K/D）、伤害输出/承受、射击命中率、血包拾取等。
    2.  **用户研究**：**15**名人类玩家在不知道对手类型的情况下，分别与三种AI进行对抗，并在5点李克特量表上对对手的挑战性、技能、协调性、享受度和人类相似度进行打分，最后选出最像人类的对手。

### 6. 论文的主要结论与发现

1.  **性能相当**：LLM+RL系统的胜率（46.4%）与手工设计的行为树（51.5%）**无统计显著差异（p=0.103）**，两者都显著优于端到端训练的Flat RL。
2.  **行为更接近人类**：用户研究中的**60%** 参与者认为LLM+RL智能体最像人类玩家，其比例显著高于Flat RL（33.3%）和BT（6.7%）（p=0.027）。用户特别提到其**行为适应性**、**战术多样性**（如在血量低时使用血包、跳跃躲闪）和**不可预测性**。
3.  **技能选择模式合理**：LLM+RL的技能选择和过渡模式与BT不同，展现出更高的策略熵（更多样化的过渡）和更明显的上下文依赖性（如低血量时选择撤退，在目标区域内选择固守）。这表明LLM能够进行更为动态和非模式化的决策。
4.  **无需手工规则**：该方法证明了通过自然语言界面，设计者只需定义技能，而将战略协调的重任委托给LLM，减少了开发复杂性。

*   **实用的“预训练-协调”范式**：避免了从头学习层次化策略的困难，通过预训练技能库并使用LLM进行编排，是工程上有效且可行的路径。

*   **方法设计完整**：对LLM提示、RL奖励函数、训练环境等细节都进行了详细描述，复现性强。

### 8. 不足与局限：包括实验覆盖、偏差风险、应用限制

*   **实验局限性**：
    *   **单一环境**：只在作者自创的一个2v2环境中进行了测试，结论在其他类型（如完全合作、大规模团队、即时策略）游戏或真实机器人场景中的泛化能力未知。

*   **技术局限**：
    *   **LLM缺乏对技能执行状态的感知**：LLM基于当前状态选择技能，但不清楚该技能在目标智能体上执行的后续效果，可能导致过早、过晚或不恰当的技能切换。

    *   **手动提示工程**：虽然减少了规则工程，但提示（prompt）本身仍需要手动设计和调试，其质量对系统性能有直接影响。
*   **应用限制**：该方法依赖于一个定义良好且可复用的技能库。如何自动发现和构建这些技能，以及在技能库庞大时如何进行选择，仍是未来挑战。
