## 摘要
多智能体目标识别要求观察者共同推断哪些智能体协同行动以及每个团队试图实现什么目标，因此假设空间随团队划分和每个团队的目标数量呈组合增长。无人机监控和协作机器人等实际应用仅暴露智能体的轨迹，这迫使观察者仅凭行为对团队-目标假设进行排序。基于分支定界的多智能体目标识别（MAGR-BB）通过将共享的团队和目标条件策略作为分解分支定界搜索内部的评分模型来解决这一设定。在受控的多智能体积木世界基准测试中，MAGR-BB在整个轨迹中返回与穷举搜索相同的顶级假设，同时将假设生成减少数个数量级，并显著降低累积识别运行时间。

## Abstract
Multi-agent goal recognition asks an observer to jointly infer which agents act together and what each team is trying to achieve, so the hypothesis space grows combinatorially with the number of team partitions and goals per team. Real applications such as drone surveillance and collaborative robotics expose only the agents' trajectory, which forces the observer to rank team-goal hypotheses from behavior alone. Multi-Agent Goal Recognition with Branch-and-Bound (MAGR-BB) addresses this setting with a shared team- and goal-conditioned policy used as the scoring model inside a factorized branch-and-bound search. On a controlled multi-agent Blocksworld benchmark, MAGR-BB returns the same top-ranked hypothesis as exhaustive search throughout the trajectory while cutting hypothesis materialization by orders of magnitude and reducing cumulative recognition runtime substantially.

## 论文提出的方法论

- **核心思想**：训练一个共享的团队‑目标条件强化学习策略（Transformer），作为评分模型；利用“非竞争性得分”性质将完整假设得分分解为各团队局部得分之和；结合因子化分支定界（Factorized Branch-and-Bound）搜索，通过多级剪枝避免枚举所有完整假设。
- **关键技术细节**：
  1. **条件策略网络**：采用 Transformer 编码器，输入包含状态特征、智能体身份、团队掩码和目标特征（反事实观测）。策略输出动作概率，支持团队内自回归解码（按智能体索引顺序）。
  2. **非竞争性得分**：每个智能体的动作对数概率仅依赖于其所在团队和团队目标，不依赖于其他团队的目标选择。因此完整假设得分为各团队局部得分之和（公式 3）。
  3. **因子化分支定界**：
     - 维护局部团队‑目标得分表（公式 2）。
     - **得分阶段剪枝**：利用过时的分区上界跳过整分区刷新。
     - **分区级剪枝**：刷新后若分区上界低于当前 top‑k 阈值，则跳过该分区。
     - **局部级剪枝**：在分区内，按目标得分降序排列，使用最佳优先堆进行目标组合搜索，当堆顶得分低于阈值时停止。
  4. **训练**：使用 PPO 算法，采用长短‑难度课程训练（长度 2‑4 的堆叠目标），集中训练‑分散执行（CTDE）。
- **算法流程**（Algorithm 1）：每步观测到达时，按过时上界排序分区 → 对每个分区判断是否需刷新 → 计算当前上界 → 若可通过分区剪枝，则进行内部目标组合搜索 → 输出当前 top‑k 堆。

## 实验设计

- **数据集/场景**：**多智能体积木世界（Blocksworld）**，受控场景：4 个智能体分为两个隐身团队（每个团队 2 个智能体），每个团队操作独立的 7 块积木工作区。目标为有序堆叠（长度 2‑4），每个团队目标库含 **1092 个候选目标**。
- **Benchmark**：完整假设空间为 6（分区数）× 1092² = 7,154,784。观察轨迹由训练好的策略滚动生成，并加入动作噪声（0.0, 0.05, 0.1, 0.2）。
- **对比方法**：6 种变体（均使用相同局部得分）：
  - Factorized Exhaustive（无剪枝）
  - Scoring Prune（仅得分阶段剪枝）
  - Partition B&B（仅分区级剪枝）
  - Local B&B（仅局部级剪枝）
  - Ranking B&B（分区+局部剪枝，无得分阶段）
  - Full B&B（完全算法，即 Algorithm 1）
- **评估指标**：最终 top‑1 团队准确率、目标准确率、联合准确率；识别延迟（首次正确步数）；计算开销（运行时间、得分刷新次数、分区访问次数、目标元组发射次数）。

## 论文的主要结论与发现

1. **精度等价**：所有六种变体在每步观测后 top‑1 假设完全一致，且最终 top‑10 列表完全一致，说明剪枝不改变排名结果。
2. **识别效率**：团队身份平均在 1.6 步后即可正确识别；联合团队‑目标识别在 6.4‑7.2 步内完成（取决于噪声）。
3. **计算节省**：
   - Full B&B 在最终步仅发射 **10 个完整假设**（穷举为 7.15M），累积发射量从 50‑60M 降至 85‑99。
   - 累积得分更新次数减少约一半（低噪声下 91.7k→46.7k）。
   - 累积运行时间加速比：低噪声 **2.91×**，高噪声 **2.43×**。
4. **噪声影响**：高噪声会削弱得分阶段剪枝效果，但 Full B&B 仍是所有测试中最快的变体。
