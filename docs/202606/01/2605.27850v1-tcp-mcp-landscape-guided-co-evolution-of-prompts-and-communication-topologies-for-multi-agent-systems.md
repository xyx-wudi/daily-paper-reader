## 摘要
有效的多智能体系统不能仅通过单独选择提示或通信图来设计。智能体行为取决于其接收的信息，而通信边的作用则取决于接收智能体如何解释和利用该信息。我们提出**TCP-MCP**（面向多智能体协作问题求解的拓扑耦合提示），这是一个协同进化框架，它将智能体提示和通信拓扑作为一个统一的基因组进行搜索。TCP-MCP在初始化阶段使用景观探测来校准早期搜索行为，然后依赖帕累托前沿诊断来在三个目标下调整探索：任务性能、令牌成本和结构复杂度。在相同DeepSeek-V3.2骨干网络下，TCP-MCP在MMLU-Pro、MMLU和GSM8K上分别达到82.66%、89.96%和96.61%的准确率。在这三个基准测试中，它始终优于自动图生成基线，并且与辩论式系统相比具有竞争力，同时在使用报告的操作点时，其令牌消耗最多减少5.69倍。这些结果表明，在受控评估中，联合进化提示和通信结构为成本感知和任务自适应的多智能体系统设计提供了一条实用途径。

## Abstract
Effective multi-agent systems cannot be designed by selecting prompts or communication graphs in isolation. Agent behavior depends on the information an agent receives, while the usefulness of a communication edge depends on how the receiving agent interprets and uses that information. We propose \textbf{TCP-MCP} (Topology-Coupled Prompting for Multi-Agent Collaborative Problem-Solving), a co-evolution framework that searches agent prompts and communication topologies as a unified genome. TCP-MCP uses an initialization-time landscape probe to calibrate early search behavior, and then relies on Pareto-front diagnostics to adapt exploration under three objectives: task performance, token cost, and structural complexity. Using the same DeepSeek-V3.2 backbone across all methods, TCP-MCP achieves 82.66\%, 89.96\%, and 96.61\% accuracy on MMLU-Pro, MMLU, and GSM8K, respectively. Across the three benchmarks, it consistently outperforms automated graph-generation baselines and achieves competitive accuracy relative to debate-style systems, while using up to 5.69$\times$ fewer tokens than those systems at the reported operating points. These results show that jointly evolving prompts and communication structure provides a practical route to cost-aware and task-adaptive multi-agent system design in controlled evaluations.

# 论文总结：TCP-MCP

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有多智能体系统（MAS）设计通常将智能体提示（prompt）和通信拓扑（communication topology）分开优化，忽略了二者的耦合效应。一个智能体的行为取决于它接收的信息，而通信边的效用取决于接收智能体如何解读和使用该信息。因此，提示和拓扑必须联合优化。
- **研究背景**：单智能体推理方法（如Complex CoT、Self-Consistency）和多智能体协议（如LLM-Debate、LLM-Blender）表现强劲，但固定拓扑结构或分离优化策略无法适应任务变化。自动化拓扑设计方法（如GPTSwarm、G-Designer）虽优化了图结构，但仍将提示视为固定组件。
- **整体含义**：本文提出将提示和拓扑作为统一基因组进行协同进化（co-evolution），以突破孤立优化带来的性能瓶颈，并在任务性能、令牌成本和结构复杂度之间取得帕累托最优权衡。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将每个MAS候选系统表示为统一基因组 M = (G, P)，其中G为有向通信拓扑图，P为各节点角色-提示对。搜索过程同时优化拓扑和提示，并使用运行时拓扑上下文动态构建提示。
- **关键技术细节**：
  - **初始化景观探针**：在初始化阶段计算Fitness Distance Correlation (FDC)，校准早期搜索偏置。
  - **结构交叉与提示最小继承（PMI）**：交换拓扑子图而非提示片段；接收节点保留原提示，移植节点继承捐赠者提示或使用轻量默认提示。
  - **变异算子**：
    - 拓扑变异：局部边/节点增删，避免形成环。
    - 角色变异：从领域角色池中更换角色，并调用PromptMutator生成新提示。
    - 激进变异：当搜索停滞时引入更大结构变化，概率由帕累托前沿诊断动态控制。
  - **环境选择与多样性保护**：采用带约束的NSGA-II，结合bin-based niching、结构配额和外部精英档案，防止早熟收敛到低成本简单拓扑。
  - **自适应控制**：基于超体积、准确性停滞、成本压力和多样性缺陷计算响应指数s，映射到激进变异概率，调节探索与利用。
- **多目标优化**：同时最大化任务准确率A(M)，最小化令牌成本C(M)和结构复杂度ln(1+K(G))，输出帕累托前沿。
