## 摘要
我们提出了AgentJet，一种用于大语言模型(LLM)智能体强化学习的分布式群组训练框架。与将智能体执行与模型优化紧密耦合的集中式框架不同，AgentJet采用解耦的多节点架构，其中群组服务器节点托管可训练模型并在GPU集群上运行优化，而群组客户端节点则在任意设备上执行任意智能体。这种设计提供了集中式框架难以支持的能力：(1) 异构多模型强化学习，能够训练以多个LLM为大脑的异构多智能体团队；(2) 具有隔离智能体运行时的多任务鸡尾酒训练；(3) 容错执行，防止外部环境故障中断训练过程；(4) 实时代码迭代，允许通过替换群组客户端节点在训练期间编辑智能体。为了支持多模型、多轮次和多智能体设置下的高效强化学习，AgentJet引入了一个带有时间线合并的上下文跟踪模块，该模块合并冗余上下文，实现了1.5-10倍的训练加速。最后，AgentJet引入了一个自动化研究系统，该系统以研究主题为输入，在大型集群上自主进行长期、多天的强化学习研究。通过利用群组架构，该系统在无需人工干预的情况下复现了强化学习研究人员的关键探索工作流程。

## Abstract
We present AgentJet, a distributed swarm training framework for large language model (LLM) agent reinforcement learning. Unlike centralized frameworks that tightly couple agent rollouts with model optimization, AgentJet adopts a decoupled multi-node architecture in which swarm server nodes host trainable models and run optimization on GPU clusters, whereas swarm client nodes execute arbitrary agents on arbitrary devices. This design provides capabilities that are difficult to support in centralized frameworks: (1) heterogeneous multi-model reinforcement learning, enabling the training of heterogeneous multi-agent teams with multiple LLM as brains; (2) multi-task cocktail training with isolated agent runtimes; (3) fault-tolerant execution that prevents external environment failures from interrupting the training process; and (4) live code iteration, which allows agents to be edited during training by replacing swarm client nodes. To support efficient RL in multi-model, multi-turn, and multi-agent settings, AgentJet introduces a context tracking module with timeline merging, which consolidates redundant context and achieves a 1.5-10x training speedup. Finally, AgentJet introduces an automated research system that takes a research topic as input and autonomously conducts long-horizon, multi-day RL studies on large-scale clusters. By leveraging the swarm architecture, this system reproduces key exploratory workflows of RL researchers without human intervention during execution.

- **问题**：现有框架存在多个局限：
  - **运行时脆弱性**：智能体 rollout 与模型训练紧密耦合，外部环境失败会导致整个训练管道崩溃。

  - **环境锁定**：多任务鸡尾酒训练要求任务间强隔离，单体架构难以实现。
- **目标**：提出AgentJet，一种分布式群组训练框架，通过完全解耦智能体运行时与训练基础设施，解决上述问题，实现灵活、可扩展、高效的智能体RL训练。

## 2. 论文提出的方法论：核心思想、关键技术细节
### 核心思想：群组架构（Swarm Architecture）
- **Swarm Server节点**：运行在GPU集群上，托管模型权重、优化器状态、vLLM/SGLang推理引擎，执行策略梯度更新和上下文跟踪。每个服务器独立训练一个模型（支持多服务器多模型）。
- **Swarm Client节点**：轻量级CPU进程，可运行在任何设备（包括无GPU笔记本），执行任意智能体工作流、读取数据集、计算奖励。客户端通过OpenAI兼容的API与服务器通信，服务器透明捕获所有上下文用于训练。
- **关键特性**：
  - 客户端可动态加入/离开，不影响训练进程。
  - 支持异构多模型训练（非共享参数）。
  - 支持多任务鸡尾酒训练（不同客户端运行不同任务，统一样本池训练）。
  - 容错执行：客户端故障可恢复或忽略，服务器继续训练。
  - 实时代码迭代：修改客户端代码后重启即可，无需重启服务器（秒级，而非分/小时）。

### 关键技术细节
- **RL训练流程**：
  1. 初始化Swarm服务器（不加载模型，等待配置）。
  2. 连接Swarm客户端（一个控制器客户端激活服务器，发送训练参数）。
  3. 收集轨迹：客户端向服务器注册episode，获得服务器特定的BaseUrl/ApiKey（编码episode ID），执行agent流程，服务器透明记录所有LLM调用的上下文。
  4. 优化模型：当样本池满足条件（如达到阈值），服务器执行上下文跟踪和时间线合并，然后进行策略梯度更新。
  5. 评估：可在任意时间/节点进行，评估episode完成后不加入训练样本池。
- **Episode Batching策略**（C1-C5）：
  - C1：基于总episode计数。
  - C2：基于完成的任务数（每个任务达到N个episode）。
  - C3：基于有效任务数（奖励方差非零的任务）。
  - C4：所有客户端同意后触发。
  - C5：任意客户端同意后触发。
- **上下文跟踪与时间线合并**：
  - 每个LLM调用生成一个时间线，包含作者标签（llm/env/user）、文本、token ID、对数概率、损失掩码。
  - **时间线合并算法（Algorithm 1）**：按逆序合并，较长的S吸收较短的T（若前缀匹配）。可选token级或text级匹配策略。文本级匹配更激进，适用于Qwen3等模型（token重编码导致漂移），可合并更多冗余，但可能牺牲训练-推理一致性。
  - **加速效果**：在AppWorld任务中，时间线合并使 actor-update 时间从2160秒降至346秒，加速6.25倍，且不损失奖励质量。

## 6. 论文的主要结论与发现
- **群组架构可行且有效**：解耦设计实现了异构多模型训练、多任务鸡尾酒训练、容错和实时调试，这些是集中式框架难以支持的。
- **时间线合并显著加速**：在不损失训练质量的前提下，actor-update时间减少1.5-10倍（AppWorld中加速6.25倍）。文本级匹配更激进，但需注意训练-推理一致性。
- **非共享参数优于共享参数**：在狼人杀中，每个狼人独立训练（非共享参数）最终胜率66.5%，高于共享参数的64.7%，归因于行为多样性。
- **鸡尾酒训练的权衡**：多任务训练可以产生一个多领域通用的模型，但在某些任务（如AppWorld）上性能低于专有模型（奖励低约10%）。鸡尾酒训练适合以低成本获得通用模型，而专有训练适合最大化特定任务性能。
- **自动化研究能力**：A3R模块能自主进行多天、多阶段的实验规划、执行和总结，成功找到AIME最小稳定batch size和最优response length组合。
- **框架无关性验证**：四种不同客户端框架（OpenAI SDK, LangChain, AgentScope, Raw HTTP）在相同设置下训练曲线几乎重合，表明AgentJet透明地支持任意LLM客户端。
- **稳定性**：跨git版本回归测试显示训练曲线高度一致，证明框架演进不影响训练质量。

## 7. 优点
- **创新的群组架构**：完全解耦训练和推理平面，突破了现有框架的耦合假设，提供了前所未有的灵活性（异构多模型、多任务、容错、实时迭代）。
- **时间线合并技术**：设计精巧，通过匹配和时间线合并消除冗余，显著加速训练，且不引入错误。
- **灵活的训练控制**：多种批次收集策略（C1-C5）适应不同需求，支持动态采样（如DAPO中的零优势丢弃）等先进方法。
- **生态兼容性**：框架无关，支持任何使用OpenAI-compatible API的智能体框架，零代码修改集成。
- **自动化研究系统（A3R）**：开创了LLM RL领域自动化实验的先河，能自主进行多天、多阶段的实验，降低人力成本，且使用开源模型驱动，成本可控。
- **实验设计充分**：包含多个真实基准和消融实验，验证了各项核心能力的有效性，定量结果令人信服。

## 8. 不足与局限
- **鸡尾酒训练的性能损失**：多任务混合训练在AppWorld上比单独训练奖励低约10%，未充分分析为什么会出现这种不对称性，也未探索不同任务混合比例的影响。

- **对比实验不足**：论文未与现有主流框架（如OpenRLHF、veRL、Forge等）进行直接的性能、资源占用或易用性对比，使得“比现有框架更好”的结论缺乏实证支持。
- **时间线合并策略的权衡**：文本级匹配可能因tokenizer差异导致训练-推理不一致（尽管论文承认但未量化影响），token级匹配则保守。用户需自己权衡，缺乏自动化指导。
- **算力细节缺失**：未报告总训练时间、GPU利用率等关键资源指标，不利于评估框架效率。
- **规模限制**：实验主要基于7B/8B/14B模型，未展示在更大模型（如30B/70B）上的训练表现，可扩展性有待验证。静态对手使用235B，但未说明其推理开销是否影响实验公平性。
- **场景覆盖**：智能体任务场景仍然有限（游戏/翻译/数学/编码），缺乏对真实世界复杂工作流（如网页导航、数据库操作）的长时间验证。
