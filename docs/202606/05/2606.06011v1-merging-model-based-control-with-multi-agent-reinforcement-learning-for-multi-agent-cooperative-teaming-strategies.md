## 摘要
在这项工作中，我们提出了一个结合多智能体强化学习（MARL）与基于模型控制的框架，以在协作多智能体任务中实现安全、动态可行的动作。多智能体强化学习具有从长期规划范围内的离散不可微奖励中学习多智能体团队协作策略的优势。模型预测控制具有鲁棒性，能在快速重规划框架下提供短时域内安全、动态可行的动作。我们提出了一种将演员-评论家模型预测控制扩展到MARL的算法，称为多智能体演员-评论家模型预测控制（MA-AC-MPC）。通过将其应用于多智能体追逃场景，我们展示了该算法的能力。具体而言，我们比较了使用MA-AC-MPC模型和多层感知机模型（MA-AC-MLP）的逃避方策略。追逐方则采用增强比例导引，因其被接受为先进的对抗控制律。我们还在异质环境（无人机与全向轮式机器人协作实现可重复成功着陆）中提供了示例，硬件上MA-AC-MPC的成功率为100%，而MA-AC-MLP仅为60%。我们在硬件上对两种环境验证了所提MA-AC-MPC算法的鲁棒性。

## Abstract
In this work, we propose a framework that combines multi-agent reinforcement learning (MARL) with model-based control to achieve safe, dynamically feasible actions in cooperative multi-agent tasks. Multi-agent reinforcement learning provides the advantage of learning cooperative policies for multi-agent teams from discrete non-differentiable rewards in a long planning horizon. Model-predictive control is robust and offers safe, dynamically feasible actions in a fast replanning framework for short horizons. We propose an algorithm that extends actor-critic model predictive control for MARL which we refer to as multi-agent actor-critic model predictive control (MA-AC-MPC). We demonstrate the capabilities of this algorithm by applying it to a multi-agent pursuit-evasion scenario. Specifically, we compare the evader team's strategy using the MA-AC-MPC model and a multi-layer perceptron model (MA-AC-MLP). The pursuer team uses augmented proportional navigation as it is accepted as an advanced adversarial control law. We also provide an example with a heterogeneous environment where a drone and omni-wheeled rover cooperate to achieve repeatable and successful landing with 100% success rate in hardware for MA-AC-MPC compared to 60% for MA-AC-MLP. We demonstrate the robustness of the proposed MA-AC-MPC algorithm in hardware for both environments.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：在集中式训练-分散式执行（CTDE）框架下，每个智能体使用一个actor-critic结构，其actor由两层组成：第一层是神经网络（成本网络），基于局部观测输出MPC的成本参数（参考轨迹和权重矩阵）；第二层是可微分MPC求解器，基于成本参数和当前状态产生动态可行的控制动作。critic则利用全局状态价值函数进行集中训练。
- **关键技术细节**：
  - **多智能体MPC策略**：每个智能体独立运行一个非线性MPC，其成本为线性最小二乘形式（参考轨迹加权），约束包含动力学、状态/控制限幅。成本参数由神经网络输出，从而将协同任务编码进MPC的优化目标。
  - **可微分MPC求解**：采用leap-c（基于acados）作为可微分MPC模块，支持通用非线性成本、动力学和不等式约束的梯度传播，比先前mpc.pytorch快且适用性更广。

1. MA-AC-MPC在追逃场景中相比同等或更大容量的MA-AC-MLP实现了更高的训练样本效率和最终胜率，尤其在质量扰动下鲁棒性显著更强（无蓝色区域，即所有质量组合下MA-AC-MPC胜率均高于MLP）。

3. 通过将低层动力学处理交由MPC，神经网络只需扮演高层任务规划器，从而降低学习难度、提高策略可迁移性与鲁棒性。

## 7. 优点

1. **方法论创新**：将可微分MPC作为actor层嵌入多智能体强化学习，实现端到端训练，同时保留MPC的可行性、安全性。
2. **工程实现优秀**：利用leap-c/acados支持通用非线性约束，加快求解速度，并兼容PyTorch/JAX。
3. **全面的实验验证**：既包含模拟（多种子、质量扫参、课程学习）也包含真实硬件（Crazyflie + ROSMASTER X3），结果具有说服力。
4. **课程学习设计**：在两种场景中均使用课程学习逐步增加难度，稳定训练过程。
5. **死亡掩码处理**：对非活跃智能体进行mask处理，避免分布偏移，提升训练稳定性。

## 8. 不足与局限

1. **计算开销**：MPC求解只能在CPU上运行，训练时间远长于纯MLP模型（162h vs 30h），限制了大规模并行加速。
2. **依赖精确动力学模型**：MPC需要显式动力学模型，对于未知物理效应或高度非线性系统可能不适用。
3. **对比基线有限**：仅对比了MA-AC-MLP，未与现有其他MARL-MPC混合方法（如基于分布的MPC多智能体方法）或更先进的纯MARL方法（如QMIX、MADDPG）比较。
4. **可扩展性未验证**：实验仅包含2个智能体（追逃）和2个智能体（着陆），未测试更大规模（如10+）的团队场景。
5. **硬件实验数量较少**：着陆场景仅各5次试验，虽然成功/失败比例差异明显，但统计显著性未明确报告；追逃硬件仅展示了单次轨迹复现。
6. **应用限制**：MPC步长固定为20ms（50Hz），规划视界短（N=2），任务对MPC实时性要求高；复杂任务可能需要更长视界，可能增加计算负担。
