## 摘要
基于LLM的多智能体系统将复杂任务分解为相互作用的角色，但大多数系统仍然通过提示、工具和控制规则进行手动编排，智能体很少通过统一的强化学习接口进行优化。现有的强化学习后训练框架主要针对单一策略优化，缺乏对用户定义的多智能体工作流、结构化交互、角色特定的信用分配和可配置参数共享的抽象。我们提出UnityMAS-O，一种基于LLM的多智能体系统的通用强化学习优化框架。UnityMAS-O将完整工作流视为优化单元，而非单个响应或策略轨迹。它通过四个一级对象表示工作流：逻辑智能体角色、图轨迹、用户定义奖励和智能体-模型映射。这将逻辑智能体与物理模型参数解耦，支持完全共享、完全分离和部分共享，并在角色、轮次和轨迹级别分配奖励。UnityMAS-O扩展了verl，采用基于Ray的星型拓扑运行时。中央控制器执行工作流、调用工具、记录结构化轨迹并组装奖励；模型本地工作线程组处理展开、缓存、优势计算和分布式PPO风格更新。用户可以定义智能体、工作流、模型映射和奖励，而无需重写优化基础设施。我们在检索增强式问答、迭代式智能搜索和反思式代码生成上实例化UnityMAS-O。在Natural Questions、HotpotQA和保留代码任务上，多智能体强化学习在优化后改进了手动指定的工作流，尤其在小模型和严格代码全部通过指标上取得了巨大提升。这些结果表明，UnityMAS-O可以作为可复用基础，将各种基于LLM的多智能体工作流转换为可训练的多智能体强化学习系统。

## Abstract
LLM-based multi-agent systems decompose complex tasks into interacting roles, but most remain manually orchestrated by prompts, tools, and control rules, while agents are rarely optimized through a unified reinforcement learning interface. Existing RL post-training frameworks mainly target single-policy optimization and lack abstractions for user-defined multi-agent workflows, structured interaction, role-specific credit assignment, and configurable parameter sharing.   We present UnityMAS-O, a general RL optimization framework for LLM-based multi-agent systems. UnityMAS-O treats the complete workflow as the optimization unit, rather than a single response or policy trajectory. It represents workflows through four first-class objects: logical agent roles, graph trajectories, user-defined rewards, and agent--model mappings. This decouples logical agents from physical model parameters, supporting full sharing, full separation, and partial sharing, with rewards assigned at role, turn, and trajectory levels.   UnityMAS-O extends verl with a Ray-based star-topology runtime. A central controller executes workflows, invokes tools, records structured trajectories, and assembles rewards; model-local worker groups handle rollout, buffering, advantage computation, and distributed PPO-style updates. Users can define agents, workflows, model mappings, and rewards without rewriting the optimization infrastructure.   We instantiate UnityMAS-O on retrieval-augmented QA, iterative agentic search, and reflective code generation. Across Natural Questions, HotpotQA, and held-out code tasks, multi-agent RL improves manually specified workflows after optimization, with especially large gains for smaller models and strict code all-passed metrics. These results show that UnityMAS-O can serve as a reusable substrate for converting diverse LLM-based multi-agent workflows into trainable multi-agent RL systems.

## 1. 论文的核心问题与整体含义（研究动机和背景）

当前基于 LLM 的多智能体系统（如 CAMEL、AutoGen、ChatDev）大多采用手工编排方式：通过提示词、工具调用和控制规则定义工作流，智能体之间的交互模式在推理时固定，缺乏统一的强化学习优化接口。现有的 RL 后训练框架（如 TRL、OpenRLHF、verl）主要针对**单一策略模型**进行优化，无法直接支持用户自定义的、包含多个交互角色、结构化轨迹和角色级信用分配的多智能体工作流。  
**核心问题**：如何构建一个通用的、可复用的优化基础设施，将任意基于 LLM 的多智能体工作流转化为可训练的 RL 系统，使研究人员能够灵活定义角色、模型映射、奖励函数，并支持完全共享、部分共享和完全分离的参数模式。  
**整体含义**：UnityMAS-O 试图填补“工作流设计”与“RL 优化”之间的鸿沟，使多智能体系统的优化不再依赖任务特定的实现。

## 2. 论文提出的方法论

### 核心思想
将 **完整多智能体工作流** 而非单个响应或策略轨迹作为优化单元。通过四个一级对象抽象工作流：
- **逻辑智能体角色**（如规划者、检索者、编码者、验证者等）
- **图结构化轨迹**（描述顺序、并行、迭代、混合执行路径）
- **用户定义的奖励函数**（可在角色、轮次、轨迹级别分配）
- **显式的智能体-模型映射**（逻辑角色与物理 LLM 实例的对应关系）

### 关键技术细节
- **解耦逻辑与物理参数**：映射 φ: V → M 支持完全共享、完全分离、部分共享三种参数体制，改变映射即可改变训练布局，无需改写工作流逻辑。
- **分布式运行时**：基于 Ray 的星型拓扑结构。一个**中央控制器**执行工作流、调用工具、记录轨迹、组装奖励；多个**模型本地工作线程组**负责推理、缓存、优势计算和 PPO 风格更新。
- **奖励接口**：每个角色可定义独立奖励函数 R_v(s, y_v, s’, τ)，支持节点级、轮次级、轨迹级信息的组合，包括延迟奖励和边际改进奖励。
- **训练流程**：
    1. 控制器异步启动任务，分发角色调用至对应工作线程组。
    2. 工作线程组返回轻量输出，模型侧保留“胖”张量（log probs、value 等）。
    3. 控制器评估奖励，提交至对应模型的缓存。
    4. 每个工作线程组构建就绪批次，计算 GAE 优势，执行 PPO 更新。
    5. 同步权重至 vLLM 后端。

### 算法流程（文字说明）
1. 用户定义逻辑角色、工作流图、模型映射和奖励函数。
2. 控制器对每个任务实例执行工作流，记录状态序列和动作。
3. 根据映射 φ 将生成请求路由到对应的 LLM 工作线程组。
4. 工作线程组生成输出，返回控制器；同时保留张量在本地缓存。
5. 控制器评估奖励（角色/轮次/轨迹级），将其与角色 ID 一起匹配到缓存。
6. 缓存足够后，每个工作线程组计算优势并执行多轮 PPO 参数更新。
7. 更新后的权重同步至推理后端，继续下一轮迭代。

1. **UnityMAS-O 能有效提升多智能体工作流性能**：在所有 QA 和代码设置中，训练后的验证 F1 或 held-out 全部通过率均优于训练前。

## 7. 优点

- **统一的抽象层**：将工作流图、角色、模型映射、奖励统一为框架级对象，使用户无需重写训练基础设施即可构建不同多智能体系统。
